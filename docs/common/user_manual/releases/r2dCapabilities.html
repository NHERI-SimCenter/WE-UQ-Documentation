CapabilitiesÔÉÅ

Version 4.4 of WE-UQ app was released in May 16, 2025, introducing significant updates and enhancements. This document outlines the functionalities available in the current version, highlighting new features and improvements in blue.

Major updates from the previous version is related to changes made to incorporate the latest release of Pelicun and the DL models. Also changes have been made to how and when the autopopulation scripts run in the workflow.

Release date: May, 2025

  1. Hazard Types:

    • User specified earthquakes.

    • User specified hurricanes.

    • ShakeMap Scenarios for earthquake events, allowing the import of ShakeMap XML grid file, PGA Contours (.json) file, and Fault Rupture (.json) file.

    • Raster Defined Hazard for Earthquake, Hurricane, Inundation.

    • GIS Defined Hazard for Earthquake, Hurricane, Inundation.

  2. Asset Types:

    • Buildings:

      1. Load building database in a .csv format.

      2. Load building database in a .gis format.

    • Transportation infrastructure:

      1. Load transportation infrastructure database in SimCenter‚Äôs .geojson format.

      2. Load transportation infrastructure database in common .gis format.

    • Water Distribution Network:

      1. Load water distribution network database in EPANet‚Äôs format (used for REWET).

      2. Load water distribution network from .csv files representing nodes and pipelines.

      3. Load water distribution network from GIS files representing nodes and pipelines.

      4. Load water distribution network database in EPANet‚Äôs format.

  3. Asset Modeling:
    • Buildings:

      1. MDOF-LU (MDOF shear building model).

      2. OpenSeesPy building generator.

      3. None (Used for the IMasEDP and Capacity Spectrum Method options in Asset Analysis).

      4. CustomPy.

    • Transportation infrastructure:

      1. None (Used for the IMasEDP option in Asset Analysis).

  4. Asset Analysis:
    • Buildings:

      1. OpenSees.

      2. OpenSeesPy.

      3. IMasEDP (simplified analysis using Intensity Measures (IMs) as Engineering Demand Parameters (EDPs)).

      4. Capacity Spectrum Method.

      5. CustomPy for CustomPy Asset Modeling.

      6. Pre-trained surrogate models.

    • Transportation infrastructure:

      1. IMasEDP (simplified analysis using Intensity Measures (IMs) as Engineering Demand Parameters (EDPs)).

  5. Damage and Loss:
    • Buildings:
      • Pelicun Damage and Loss Methods:

        1. HAZUS MH EQ Story

        2. HAZUS MH EQ IM

        3. HAZUS MH HU

        4. User-provided Models

    • Transportation infrastructure:
      • Pelicun Damage and Loss Methods:

        1. HAZUS MH EQ IM

        2. User-provided Models

    • Water Network infrastructure:
      • Pelicun Damage and Loss Methods:

        1. HAZUS MH EQ IM

        2. User-provided Models

  6. System Performance:

    • REWET System performance for water network with or without recovery

    • ResidualDemand Post-disaster traffic flow simulation

  7. Recovery:

    • Regional Recovery simulation using Pyrecodes.

  8. Uncertainty Quantification:
    • Dakota:

      1. Latin hypercube sampling (LHS)

      2. Monte Carlo Sampling (MCS)

  9. Additional Tools To Perform Tasks Generating or Using Data in Workflow:
    • Earthquake Scenario Simulation (ground motion selection)
      • Site definition:

        1. Grid

        2. Point

        3. Scattered sites (user-defined sites in .csv format)

      • Rupture forecast models:

        1. OpenSHA UCERF rupture forecast models

        2. OpenSHA Point source user-defined

        3. OpenQuake rupture forecast

        4. Hazard Occurrence Model

      • Inter-event correlation:

        1. Baker and Jayaram (2008)

      • Intra-event correlation:

        1. Jayaram and Baker (2009)

        2. Markhvida et al. (2017)

        3. Loth and Baker (2013)

      • Record selection:

        1. PEER NGA West 2 ground motion database

        2. None, i.e., stop at the IM stage and no record selection

      • Ground motion models:

        1. Abrahamson, Silva & Kamai (2014)

        2. Boore, Stewart, Seyhan & Atkinson (2014)

        3. Campbell & Bozorgnia (2014)

        4. Chiou & Youngs (2014)

      • Intensity measures:

        1. Spectral acceleration (SA)

        2. Peak ground acceleraation (PGA)

        3. Peak ground velocity (PGV)

      • Ground failure models:
        1. Liquefaction triggering

          1. Zhu et al. (2017)

          2. Hazus (2020)

        2. Liquefaction lateral spreading permanent ground deformation (PGD_h)

          1. Hazus (2020)

        3. Liquefaction settlement permanent ground deformation (PGD_v)

          1. Hazus (2020)

      • Vs30 model:

        1. CGS/Wills Vs30 (Wills et al., 2015)

        2. Thompson California Vs30 (Thompson et al., 2018)

        3. Global Vs30 (Heath et al., 2020)

        4. User Defined

    • Hurricane Scenario Simulation (hurricane wind field generation)

        1. Grid

        1. User-defined sites in .csv format

        2. Select from a database of historical hurricanes

        3. Truncate hurricane track functionality

      • Landfall location and parameters:

        1. User selects on GIS map

        2. Manual user entry in the input box

      • Wind field generation model:

        1. Snaiki and Wu (2017)

    • OpenQuake Selection

    • Census Data & American Community Survey

    • Building and Infrastructure Recognition using AI at Large-Scale (BRAILS)

      • Building inventory generation
